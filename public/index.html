The 7 Pillars of D.O.R.I.S.

Every pillar is optional. Add or remove bindings as your app grows.

01

Rust вҶ’ WASM via worker-build

Rust compiles to WebAssembly using the worker crate and worker-build. Axum runs inside the Worker for ergonomic routing. Pin the toolchain for reproducible CI builds.

# rust-toolchain.toml
channel = "1.91.1"
targets = ["wasm32-unknown-unknown"]
02

JS Entry Wrapper

Thin JS layer (src/entry.js) imports the Rust WASM module, routes WebSocket connections to NotifyDo, CPU-heavy endpoints to HeavyDo, and falls through everything else to Rust.

if (env.HEAVY_DO) {
  const stub = env.HEAVY_DO
    .idFromName(shardKey);
  return stub.fetch(request);
}
return worker.fetch(request);
03

HeavyDo CPU Offload

Durable Objects have a 30s CPU budget vs Workers' few ms. HeavyDo reuses the same Axum router вҖ” no duplicated business logic. Handles: imports, password hashing, key rotation, 2FA, account deletion.

impl DurableObject for HeavyDo {
  async fn fetch(...) {
    // Same router as main worker!
    router::api_router(self.env.clone())
  }
}
04

NotifyDo WebSocket Push

Tags-based fan-out for real-time sync. Each WebSocket connection tagged by user ID. Anonymous hub for file downloads. Optional mobile push via relay service.

// Tag a connection
ws.tag("user:abc123");
// Fan out to all user tags
ws.broadcast("user:abc123", msg);
05

D1 Database

SQLite-compatible, serverless, auto-scaling. Session-backed reads with first-primary consistency for write-then-read. Unified Db enum with d1_query! macro for ergonomic prepared statements.

[[d1_databases]]
binding = "vault1"
database_name = "vault1"
database_id = "${D1_DATABASE_ID}"
06

Rate Limiting + Assets

Cloudflare-native rate limiting via [[ratelimits]]. Static assets served directly from Cloudflare's edge. run_worker_first limits Worker invocation to API routes only. Secrets via Cloudflare Dashboard.

[[ratelimits]]
name = "LOGIN_RATE_LIMITER"
simple = { limit = 5, period = 60 }
07

One-Click CI/CD

3 GitHub Actions workflows: Build (main вҶ’ prod), Deploy Dev (dev branch), Backup D1 (daily cron). Idempotent D1 bootstrap. Fork вҶ’ set 3 secrets вҶ’ push main вҶ’ auto-deploys.

# D1 bootstrap (idempotent)
if DB empty:
  apply schema.sql
  mark migrations
apply pending migrations
npx wrangler deploy

Key Files

src/entry.js

JS wrapper that routes WebSocket вҶ’ NotifyDo, CPU-heavy вҶ’ HeavyDo, everything else вҶ’ Rust WASM.

src/durable/heavy_do.rs

~60 lines вҖ” just calls router::api_router() like the main Worker. No duplicated logic.

wrangler.toml

Dual env (prod/dev), DO bindings, D1, KV, rate limits, static assets, cron triggers.

.github/workflows/push-cloudflare.yaml

Full one-click deploy pipeline with D1 bootstrap resilience.

Quickstart Checklist

1

Create from template

Clone warden-worker or scaffold from scratch with the project structure above.

2

Set up Rust + WASM

rustup target add wasm32-unknown-unknown + install worker-build

3

Write wrangler.toml

D1 binding, DO bindings, rate limits, asset directory, cron triggers.

4

Create D1 database

npx wrangler d1 create my-db

5

Build your Axum router

Define all routes in src/router.rs, the single source of truth.

6

Add Durable Objects

HeavyDo for CPU offload + NotifyDo for WebSocket push in src/durable/

7

Set up CI/CD

3 workflows: build+deploy (main), deploy dev (dev branch), D1 backup (daily cron).

8

Deploy рҹҡҖ

Add GitHub Secrets вҶ’ push to main вҶ’ auto-deploys to Cloudflare.
